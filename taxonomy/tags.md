# 标签体系

本仓库使用多标签组织资料，目录只负责轻量归档。

## object_type：资料本体

- `open_source_project`：开源系统 / GitHub 项目
- `article`：文章 / 长文 / 经验分享
- `trend_signal`：行业趋势 / 观察判断
- `commercial_product`：商业产品 / SaaS / 工具
- `case_or_media`：案例 / 视频 / 自媒体内容
- `methodology`：方法论 / 框架 / 课程型观点

## source_type：发现来源

- `GitHub`
- `YouTube`
- `小红书`
- `公众号`
- `官网`
- `论文`
- `群聊线索`
- `播客`
- `新闻`

## business_tags：业务对象

- `市场`
- `销售`
- `产品`
- `运营`
- `管理`
- `客服`
- `供应链`
- `ITBP`
- `个人能力`

## problem_tags：问题域

- `获客`
- `转化`
- `客户经营`
- `流程提效`
- `知识沉淀`
- `用户洞察`
- `组织协同`
- `内容生产`
- `品牌增长`
- `MVP验证`
- `需求分析`

## method_tags：方法 / 能力

- `Agent`
- `GEO`
- `Vibe Coding`
- `AI Coding`
- `RPA`
- `Prompt`
- `Skill`
- `自动化`
- `知识库`
- `数据分析`
- `Playbook`

## value_stage：价值阶段

- `学习理解`：主要用于补认知、补方法、补背景
- `待追源`：需要继续找原始资料、本体、代码或文档
- `待验证`：已初筛，但需要实测或业务确认
- `可小实验`：适合做低成本验证
- `可业务试点`：具备进入业务试点的可能
- `已沉淀`：已转为模板、方法、Playbook 或稳定经验
- `暂不建议`：当前不适合投入

## risk_tags：风险边界

- `版权`
- `数据安全`
- `国内可用性`
- `权限`
- `成本`
- `幻觉`
- `合规`
- `夸大包装`
- `维护风险`
- `组织责任`

## public_level：公开级别

- `public`：可公开
- `sanitized`：脱敏后可公开
- `private-only`：仅内部保留，不进入公开仓库
