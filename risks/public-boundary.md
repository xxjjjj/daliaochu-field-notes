# 公开边界

## 原则

公开的是判断框架和二次研究，不公开内部信息，也不搬运别人的原始内容。

## 可以公开

- 外部公开链接
- 自己写的摘要、评论、判断和业务转译
- 通用模板、标签体系、评估方法
- 不涉及公司敏感信息的实验记录
- 对公开工具、开源项目、商业产品的试用感受

## 不公开

- 公司内部群聊原文
- 客户信息、系统截图、业务数据、真实流程细节
- 未脱敏的 CRM / 飞书 / NC / RPA / 内部系统实践
- 付费课程内容、完整字幕、逐字稿、完整翻译
- 未授权课件、专有资料、受版权保护的大段原文
- 未验证的收益数字和夸张营销结论

## 技术 / 开源资料处理规则

看到自媒体内容时，只把它当作发现入口。真正结论必须尽量追到：

- GitHub 仓库
- 官方网站
- 官方文档
- Demo
- 论文
- 作者说明

未看到本体前，只能写“初步信号”，不能写“推荐使用”。

## 企业场景处理规则

涉及企业内部实践时，只公开抽象方法，不公开真实业务细节。

可以写：

- 某类流程如何拆解
- 某类系统如何评估
- 某类岗位如何使用 AI
- 通用风险和检查清单

不要写：

- 具体客户
- 具体组织和人员
- 真实审批链路
- 内部字段、接口、截图、日志
- 未公开业务策略
