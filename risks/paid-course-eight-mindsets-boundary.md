# 付费课程「八大心智」公开边界

## 背景

打捞处历史补录过程中，可能会遇到与付费课程「八大心智」相关的资料、纪要、讨论和实践反馈。这类内容存在明确版权和授权风险，不应进入公开仓库。

## 结论

「八大心智」相关课程内容默认标记为：

- public_level: private-only
- value_stage: 暂不建议公开
- risk_tags: [版权, 合规, 付费课程]

不得上传到公开 GitHub 仓库的内容包括：

- 课程原文、逐字稿、完整纪要；
- 课件结构、章节体系、专有表达；
- 老师原话、案例原貌、课堂问答；
- 能替代课程交付的总结笔记；
- 未授权的图片、截图、录音、视频、文件。

## 可以保留的处理方式

如确实需要沉淀，只能做以下低风险处理：

1. 在本地或私有库记录“有版权风险，暂不公开”。
2. 在公开仓库只保留风险边界规则，不保留课程内容本身。
3. 将企业内部实践转成完全脱敏、二次创作的通用管理反思，但不得复刻课程结构和专有话术。
4. 如果后续要公开，需要先确认授权范围。

## 历史补录规则

在历史消息补录时，如识别到以下线索，应直接跳过公开入库：

- 八大心智；
- 付费课程；
- 英领课程原文、逐字稿、课件、老师原话；
- 课程截图、录音、视频、飞书智能纪要；
- 可还原课程内容的系统化总结。

可以在补录工作台中标记为 `private-only` 或 `needs-review`，但不得将具体内容写入公开 notes、experiments 或 playbooks。
