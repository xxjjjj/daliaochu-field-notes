# EDM 工具命名混淆：msmtp / Focussend / 询盘云 CRM 的边界

## 1. 这是什么

打捞处群 2026-06-30 用户问"有个 EDM 的工具我记得安装了"，
bot 核查发现**当前机器里没有名为 `edm` 的命令行工具**，
但相关线索有 3 条，分别属于不同层：

| 名称 | 是什么 | 凭证位置 | 适用场景 |
|------|--------|----------|----------|
| `msmtp` | 本机命令行 SMTP 发信工具 | 本机配置 + 邮箱密码/API key | 给脚本/Agent 做"发邮件"能力 |
| `mailx` | 传统命令行发邮件工具 | 同上 | 基础邮件发送 |
| 询盘云 CRM 内置 EDM | 业务系统功能 | 询盘云 CRM 账号 | 业务侧的营销邮件 |
| Focussend EDM | 第三方 EDM 服务 | Focussend 账号 + API key | 营销自动化 / 大批量发信 |

**用户记忆模糊的"EDM 工具"，大概率不是单一工具，而是这 4 个的混合记忆。**

## 2. 为什么会出现这种混淆

- **命名层**：中文"EDM"是营销术语（Email Direct Marketing），
  对应到本机可能叫 `msmtp` 或 `mailx`，对应到业务系统可能是"询盘云 EDM 功能"
  或"Focussend 服务"。**同一个中文词对应 4 个不同工具**。
- **记忆层**：用户日常接触多个工具，时间长了容易混在一起。
  这种"我记得装过一个 X 工具"的话，背后可能是 1~3 个工具的总和记忆。
- **凭证层**：每个工具都有独立的账号/凭证/API key，
  命名混淆会导致**离职人员凭证清点时漏掉**（参见 2026-05-28 离职人员资源清点话题）。

## 3. 风险边界

- **凭证盘点遗漏**：如果把"EDM 工具"当一个东西去清点，会漏掉其中 3 个的凭证。
- **错误归属**：把营销邮件的发送行为归到本机 msmtp，会触发企业邮箱外发审计告警。
- **重复采购**：业务方不知道"询盘云已经包含 EDM"，
  会单独采购 Focussend，导致重复付费。
- **离职交接**：离职人员如果用过这 4 个工具中的任何一个，
  凭证清单上必须分别列出，不能合并。

## 4. 最小可执行的盘点流程

1. **列出本机所有发邮件相关命令**：`msmtp`、`mailx`、`sendmail`、`postfix` 等
2. **列出业务系统里的"邮件/EDM"功能模块**：询盘云 CRM、SaaS 营销系统
3. **列出第三方 EDM 服务订阅**：Focussend、Mailchimp、SendGrid 等
4. **交叉核对人员-工具关系表**：每个离职/在岗人员分别用到哪些
5. **凭证清点**：每个工具分别列出账号/API key/refresh_token 位置

## 5. 给 Agent 的判断启发

- 用户说"我装过 X 工具"时，**不要默认 X 是单一工具**，
  先问"你记得是做什么用的吗？是命令行还是网页？"
- 工具盘点类问题，应该先做"分层命名清单"再回答，
  而不是直接给一个答案。
- 涉及凭证/账号的命名混淆，必须用清单 + 表格，
  而不是文字描述。
