## 4. 业余 / 中小企业 IT 团队友好的筛选标准（公开版）

不依赖任何内部信息，任何人可拿去用：

1. **线上为主**：省出差与时间成本。
2. **无年龄 / 学生身份限制**：避免中年从业者被刷掉。
3. **Demo 门槛适中**：能现场跑通，不要求完整商业系统。
4. **主题与企业真实场景可对齐**：避免作品变成纯概念 PPT。
5. **评委重视业务价值 > 技术炫技**：经验型选手有优势。
6. **截止时间 1-2 周之后**：留足时间做脱敏和设计 Demo。

## 5. 通用参赛前自检清单

### 5.1 资格 / 合规
- [ ] 是否需要 KYC？是否能用国内身份？
- [ ] 是否需要 VPN / 海外电话 / 海外信用卡？
- [ ] 是否限制国籍？是否限制 18-30 岁？
- [ ] 主办方所在司法管辖区对中国团队是否友好？

### 5.2 主办方诉求
- [ ] 主办方是谁？他们的核心商业利益是什么？
- [ ] 是否要求落地到指定平台（绑定风险）？
- [ ] 奖金发的是法币、稳定币、平台币、链上币？
- [ ] 是否会要求参赛作品的 IP 归属 / 后续商业权益？

### 5.3 团队可投入
- [ ] 团队人数是否合适（多数黑客松限 1-4 人）？
- [ ] 周末 + 工作日晚上是否能凑出 1-2 周集中时间？
- [ ] 是否有现成业务场景可包装成 Demo？

### 5.4 风险 / 退出成本
- [ ] 提交作品后是否会被公开？是否会要求开源？
- [ ] 退出 / 中途放弃是否有违约金 / 名誉风险？
- [ ] 奖金到手是否需要缴税？是否需要特殊报税路径？

## 6. 主办方"想白嫖什么"判断信号

来自公开讨论中识别出的判断信号（不一定 100% 准确，但有参考价值）：

| 信号 | 含义 |
|------|------|
| 主办方明确要求"基于 X 平台 / X 模型" | 想拉用户 / 想用你的作品当案例 |
| 主办方明确要求"开源 + 商用授权" | 想借参赛拿低成本代码资产 |
| 主办方明确要求"赛后在 X 平台运营 30 天" | 想拿真实流量 + 真实业务数据 |
| 奖金 > $100K + 多人队伍上限 < 4 | 主办方大概率是"小成本大传播"打法 |
| 不限年龄 / 不限国籍 / KYC 简单 | 主办方要规模，没空精细审核 |
| 限定生态链上 / 限定某平台部署 | 主办方在用奖金换产品 adoption |

## 7. 对 BP / ITBP 的实用价值

- **市场部**：把"参赛 + Demo"作为对外内容素材来源（比赛过程 = 天然流量话题）。
- **产品部**：用参赛做"低成本 MVP 验证"，避免自掏腰包开发。
- **销售部**：把获奖作品变成"客户案例 + 信任背书"。
- **HR**：把参赛作为"雇主品牌"和"AI 氛围建设"动作。
- **ITBP 自身**：把"参赛 + 赛后复盘"作为 IT 团队 AI 能力建设载体。

## 8. 可做的小实验

- 列出公司近 6 个月内的真实业务场景（销售易 / 询盘云 / CRM / 知识库 / RPA）。
- 对照上面"标准"和"自检清单"，筛出 1-2 个最适配的黑客松。
- 组 3-4 人小队，用公司真实场景做脱敏 Demo 参赛。
- 不管是否获奖，把参赛过程沉淀为内部可复用的"AI 业务落地 Demo 模板"。

## 9. 风险和边界

- **数据安全 / 合规**：用真实业务场景参赛必须做脱敏；客户数据、合同数据、内部账号必须打码或抽象化。
- **国内可用性**：部分黑客松需 VPN / 海外账号；部分 KYC 不接受国内身份。
- **版权 / 作品归属**：部分比赛要求提交后开源或授予主办方使用权，参赛前必须读协议。
- **主办方绑定**：把参赛作品当主推产品前，确认主办方对作品的"商业权益主张"边界。
- **中年选手年龄限制**：部分学生向比赛有年龄 / 学历硬性要求。
- **过度包装**：公开赛道总览不夸大奖金、不夸大获奖概率；只做"匹配度判断"，不做"承诺"。

## 10. 当前结论

0G APAC / Qwen Cloud / AdventureX / ETHGlobal 是 2026 H1 业余 + 中小企业
IT 团队可重点关注的几条赛道。挑选标准比"哪场奖金高"更重要——选错了
即使获奖也可能被绑定 / 被白嫖 / 被合规卡死。

落地策略：先选 1 场做参赛实验，沉淀"参赛 + 脱敏 + Demo + 复盘"模板，
再考虑要不要常态化把"黑客松参赛"作为 IT 团队能力建设动作。
