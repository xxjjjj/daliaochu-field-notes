# 打捞处 Field Notes

> 不收藏热点，只打捞价值。  
> 从外部 AI 信号，到学习理解、真实性验证、业务转译与企业落地。

这是一个公开的 AI 学习与业务转译仓库，用来记录外部 AI 工具、开源项目、商业产品、行业观点、案例和方法论，并评估它们对个人成长与企业 AI 落地的真实价值。

## 这个仓库不是

- 不是 AI 工具收藏夹
- 不是自媒体内容搬运库
- 不是付费课程/字幕/逐字稿归档
- 不是公司内部方案公开区
- 不是看到热点就直接推荐使用

## 这个仓库是

一个从外部信号到价值转化的公开研究系统：

```text
外部信号
  ↓
追原始来源
  ↓
学习理解
  ↓
真实性 / 可用性验证
  ↓
业务转译
  ↓
小实验 / 试点判断
  ↓
沉淀为方法、模板、Playbook、索引
```

## 核心问题

每条打捞笔记尽量回答：

1. 这是什么？
2. 原始来源在哪里？
3. 它讲的核心观点或能力是什么？
4. 我学到了什么？
5. 它是否可信，哪些地方需要验证？
6. 对个人能力有什么价值？
7. 对企业 AI 落地有什么价值？
8. 可以做什么低成本实验？
9. 风险和边界是什么？
10. 当前结论是什么？

## 组织方式

本仓库采用“时间归档 + 多标签 + 索引视图”，不强行用单一目录分类。

```text
notes/        # 所有打捞卡，按时间归档
indexes/      # 按业务、方法、工具、状态等视角整理索引
templates/    # 打捞卡、工具评估、小实验模板
taxonomy/     # 标签体系
experiments/  # 小实验和验证记录
playbooks/    # 已沉淀的方法、流程、业务转译卡
risks/        # 版权、数据安全、公开边界
essays/       # 阶段性思考文章
```

## 公开边界

公开的是：

- 外部公开链接
- 自己写的摘要、评论和判断
- 通用方法论、模板和标签体系
- 脱敏后的实验记录
- 不涉及公司敏感信息的业务转译

不公开的是：

- 公司内部群聊原文、客户信息、系统截图、真实流程细节
- 付费课程内容、字幕、逐字稿、完整翻译
- 未授权课件、专有资料
- 未脱敏的内部实践
- 未验证的夸张收益结论

详见：[`risks/public-boundary.md`](risks/public-boundary.md)
