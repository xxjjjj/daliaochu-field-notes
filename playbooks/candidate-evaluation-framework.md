# 候选人评估框架 Playbook

## 1. 它是什么

面向「实施 + 数据」方向应届生 / 初级候选人，把「值不值得培养 / 潜力在哪 / 风险在哪 / 未来路径适配」四个维度的判断沉淀为可复用模板，配套「聊天式面试官脚本」生成方法。

本卡只沉淀方法论与脚本骨架，不包含任何真实候选人姓名、背景、评估结果。所有真实评估记录与候选人材料一律不入公开仓库。

## 2. 解决什么问题

- 每次看新简历都从零起判断框架，容易漏维度、主观漂移。
- 团队内部对「什么是好苗子」标准不一致，靠面试官个人发挥。
- 评估完后输出两份内容（评估 + 面试官脚本），写法和详略差异很大，不利于横向对比和长期复盘。
- 涉及真实候选人时，公开 / 半公开渠道分享评估文本存在隐私与合规风险。

## 3. 四维度判断骨架

### 3.1 维度一：值不值得培养

判断候选人的**底层学习能力 + 自驱力 + 反馈接收方式**，而不是当前技能匹配度。

**评估锚点**：

- 学业 / 项目中有没有「超出要求」的自选动作
- 面对不熟悉领域时，是查文档 / 找人问 / 还是硬上
- 面试中遇到不会的题，是直接放弃 / 乱猜 / 还是讲思路
- 自我复盘意识：有没有主动总结过自己做错的 / 学到的事

**红牌信号**：

- 简历里写「精通 X」但讲不出 X 的核心取舍
- 答不上来就沉默，不解释也不会引导
- 所有项目都是「完成作业」风格，没有自发探索痕迹

### 3.2 维度二：潜力在哪

判断候选人**在团队未来 1-2 年路径上的可塑性**，重点看「能力迁移速度」和「未来能扛什么级别的事」。

**评估锚点**：

- 跨学科背景：统计 + 实施，CS + 业务，理工 + 内容
- 已展示的「小影响力」：哪怕只是组内分享、开源小项目、社群运营
- 学习新工具的速度：现场观察（不能光听他说）
- 对「自己 3 年后想做什么」有没有哪怕模糊的答案

**绿牌信号**：

- 有从 0 到 1 的完整小闭环经验（不是只做某一段）
- 会主动记录方法 / 模板 / 笔记
- 对自己不知道的事不装懂，会主动问

### 3.3 维度三：风险在哪

判断候选人**可能踩的坑 / 文化冲突 / 稳定性 / 价值观差异**。这一维度不是为了筛掉人，是为了提前做预期管理。

**风险分类**：

| 类型 | 关注点 | 提问方式 |
|---|---|---|
| 稳定性 | 城市 / 家庭 / 通勤 / 经济压力 | 「未来 2-3 年在哪生活」「是否接受偶尔出差」 |
| 文化冲突 | 沟通风格 / 反馈接收 / 团队偏好 | 「上次跟同学 / 同事有分歧你怎么处理」 |
| 价值观 | 对加班 / 压力 / 失败的态度 | 「讲一次你放弃 / 失败的经历，你怎么复盘」 |
| 技能断层 | 简历上写的能力 vs 实际深度 | 「X 工具你能不能现场跑一下」 |
| 期望错配 | 薪资 / 职级 / 成长速度预期 | 间接问 + 主动告知团队现状 |

**特别留意**：

- 应届生最常见的不是能力问题，是「城市 / 房租 / 感情 / 家庭」导致的稳定性风险
- 「什么都想做」的人，可能不是潜力大，是没想清楚

### 3.4 维度四：未来路径适配

判断候选人**和团队未来 1-2 年要走的路径是否对得上**。这里要把团队方向摆出来对照，不要只评候选人单边。

**对照清单**：

- 团队未来 12 个月最缺什么能力？（写出来 3 条）
- 候选人 3 个最大特质分别能补上哪一条？
- 候选人最匹配的岗位画像是什么？（实施 / 数据 / 业务分析 / 内部 IT）
- 如果入职，需要补什么 onboarding？（写出来 3 条具体动作）

**输出形态**：

- 一句话定性：「3 颗星匹配 / 2 颗星匹配 / 1 颗星匹配 / 不匹配」
- 一段说明：为什么是这个星级，不是更高也不是更低
- 一句建议：建议推进 / 建议二面 / 建议储备 / 建议放弃

## 4. 面试官脚本生成模板

### 4.1 总体原则

- **偏聊天式**：应届生面对压力面容易紧张，聊天的语气能拿到更真实的信息
- **5-7 个主线问题 + 灵活追问**：不要超过 7 个，否则候选人后期会进入应答模式
- **每个主线问题 8-15 分钟**：留足时间让候选人讲完自己的故事，再追问细节
- **必问的 3 个问题**（无论简历什么样）：
  1. 讲一个你最近自己主动学的、不为学分不为工作的东西
  2. 讲一次你和别人合作 / 分歧的经历，你做了什么
  3. 你未来 2-3 年想成为什么样的人 / 做什么样的事

### 4.2 主线问题池（按四维度归类）

**维度一（值不值得培养）**：

- 你简历上写的项目 X，能不能讲一个你最得意的细节？
- 那个项目里卡住你时间最长的问题是什么？你最后怎么解决的？
- 你觉得自己学东西快不快？给一个具体的例子

**维度二（潜力在哪）**：

- 你有没有主动给别人讲过什么东西？哪怕是室友 / 学弟妹 / 社团
- 你最近一次觉得自己「突然开窍」是什么时候？是什么触发的？
- 如果给你 3 个月自由时间，没有任何考核，你会做什么？

**维度三（风险在哪）**：

- 讲一次你和朋友 / 同学 / 同事有分歧的经历，后来怎么收场？
- 你最有压力的一次经历是什么？那次你怎么撑过来的？
- 你对加班、突发任务、出差怎么看？

**维度四（未来路径适配）**：

- 你为什么投我们这个方向？你对这个方向的理解是什么？
- 你觉得自己 1 年后、3 年后应该是什么状态？
- 你觉得自己的短板是什么？如果入职你会怎么补？

### 4.3 追问模板

| 候选人回答 | 追问方向 |
|---|---|
| 「我做过 X」 | X 的具体规模 / 数据量 / 你负责的范围 / 当时的限制条件 |
| 「我和别人合作」 | 你在其中是什么角色 / 别人做了什么 / 你做了什么 / 结果如何 |
| 「我学得很快」 | 学多久 / 怎么学的 / 怎么判断自己学会了 / 用在哪 |
| 「我对这个方向很感兴趣」 | 你为这个兴趣做过什么 / 看过什么 / 投过什么 / 想清楚什么 |
| 「我愿意加班」 | 上次加班最晚的一次是什么时候 / 那次为什么 / 你怎么看加班效率 |
| 「我抗压能力强」 | 压力最大的一次 / 你做了什么 / 现在回想当时处理得对吗 |

### 4.4 评估记录表（脱敏骨架）

```text
候选人编号：
评估日期：
评估人：

[维度一] 值不值得培养
- 锚点 1：
- 锚点 2：
- 红牌 / 绿牌：

[维度二] 潜力在哪
- 跨学科背景：
- 小影响力：
- 现场学习速度：
- 未来愿景清晰度：

[维度三] 风险在哪
- 稳定性：
- 文化冲突：
- 价值观：
- 技能断层：
- 期望错配：

[维度四] 未来路径适配
- 团队最缺的能力 X / Y / Z：
- 候选人匹配哪几条：
- 入职后需 onboarding 的 3 件事：
- 星级评定（1-3 星）：
- 一句话定性：
- 下一步建议：推进 / 二面 / 储备 / 放弃
```

> ⚠️ 真实姓名、毕业院校、过往公司、薪资数字、家庭情况、感情状态一律不上表。
> 评估记录只保留「行为事实 + 判断依据」，不保留「身份信息」。

## 5. 使用方式

1. **看新简历前**：先打开本卡 §3，对照 4 维度逐条判断，避免被简历包装带偏。
2. **面试前**：从 §4.2 选 5-7 个问题，按维度交叉，不要集中在单一维度。
3. **面试中**：用 §4.3 追问表追问，不要简单问「还有吗」。
4. **面试后**：用 §4.4 评估表记录，只写行为事实和判断依据，**不写身份信息**。
5. **横向对比**：多个候选人之间，按 §3.4 的星级 + §4.4 的定性横向看，不只看单个人。

## 6. 边界与风险

- **本卡是方法论**，不绑定任何具体候选人。任何真实候选人材料、评估原文、面试记录都不入公开仓库。
- 涉及真实候选人时使用本卡，仅作为「内部 HR / 团队负责人 / ITBP」之间的私下协作工具，不对外发送。
- 应届生评估最大的坑不是「选错人」，而是「选了一个不适合团队现状的优秀年轻人」。建议在 §3.4 把团队现状写清楚，再做匹配判断。
- 如果团队未来路径本身没想清楚，建议先暂停招人，先想清楚方向。
