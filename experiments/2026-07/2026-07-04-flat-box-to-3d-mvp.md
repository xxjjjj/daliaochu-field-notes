# 平面盒型图生成 3D 立体图 MVP

## 1. 测试背景

围绕 TRELLIS / TRELLIS.2 的 3D 生成能力，群内提出一个更具体的业务问题：如果已有盒子的平面图，是否可以生成盒子的立体图。

这个问题比“普通图片转 3D”更接近包装、产品展示、销售资料和设计沟通场景，适合做一个低风险 MVP。

## 2. 测试假设

可以生成，但效果取决于输入图类型：

1. 如果输入是盒子成品的正面/斜视图，image-to-3D 模型更容易生成近似立体外观。
2. 如果输入是包装展开平面图、刀版图、六面贴图或设计稿，通用 image-to-3D 模型未必能理解“折叠关系”，可能需要额外步骤：先把平面设计映射到标准盒型模板，再生成或渲染 3D。
3. 对包装盒场景，最稳定的路线可能不是纯 3D 生成，而是“盒型参数/模板 + 平面贴图 + 渲染器/3D查看器”。AI 生成可用于补图、生成展示角度、快速出概念图。

## 3. 测试对象

- 公开、无敏感信息的盒子平面图或包装设计样张。
- 同一盒子的正面图、斜视图、展开图各 1 张，用于比较不同输入对生成结果的影响。

## 4. 测试步骤

1. 准备公开样例，不使用客户图、内部产品图或未公开包装设计。
2. 分三类输入测试：
   - 单张正面图。
   - 单张斜视图。
   - 包装展开平面图/刀版图。
3. 分别使用 TRELLIS / TRELLIS.2 Demo 或本地环境生成 glb / mesh。
4. 记录是否能形成盒子结构、贴图是否正确、边角是否变形、比例是否可信。
5. 如展开图效果差，再验证“标准盒型模板 + 贴图渲染”的替代路线。

## 5. 输入材料

待准备：公开盒子图片、公开包装展开样张、公开设计稿。

## 7. 失败点 / 异常点

预期风险：

- 模型把平面展开图当作普通图案，而不是可折叠结构。
- 盒子六面贴图位置错乱。
- 边缘、厚度、比例、开口结构不稳定。
- 医疗产品包装若用于正式展示，可能造成规格或结构误导。

## 8. 是否值得继续

值得继续，但目标应先限定为“概念展示 / 初稿沟通 / 市场素材辅助”，不要直接定位为工程级包装建模或正式产品图替代。

## 9. 下一步动作

1. 找 3 个公开包装盒样例。
2. 分别测试正面图、斜视图、展开图输入。
3. 对比 TRELLIS 路线和“盒型模板 + 贴图渲染”路线。
4. 输出一份可复用判断：什么输入适合 AI 生成，什么输入更适合模板渲染。
